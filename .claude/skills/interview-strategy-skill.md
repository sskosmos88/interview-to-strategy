# Skill: interview-to-strategy

## Назначение

Этот skill позволяет агенту (Claude) работать с инструментом `interview-to-strategy` — CLI-генератором маркетинговой стратегии из сырой расшифровки стратегического интервью со специалистом любой ниши.

Инструмент превращает текстовую транскрипцию MP3-интервью в структурированный Markdown-документ, который можно передать специалисту. Документ включает:

- сводку за 2 минуты;
- аналитику активов и пробелов;
- позиционирование и аудиторию;
- готовые тексты для площадок и контента;
- план первых шагов и метрики.

## Когда использовать

- После проведения стратегического интервью со специалистом.
- Когда есть сырой текст транскрипции, но нет времени вручную собирать стратегию.
- Когда нужно быстро выдать клиенту понятный рабочий документ.

## Артефакты

- **Вход:** `.txt` файл с расшифровкой интервью.
- **Выход:** `.md` файл стратегии, готовый к передаче клиенту.
- **Репозиторий:** `https://github.com/sskosmos88/interview-to-strategy`

## LLM-провайдеры

Инструмент поддерживает три источника анализа:

1. **Ollama** — локальная модель, бесплатно, приватно.
2. **Anthropic Claude** — высокое качество, требует API-ключа.
3. **Perplexity** — OpenAI-совместимый API, требует ключа.
4. **Fallback** — локальный rule-based анализ без LLM, результат слабее.

## Как пользоваться

### 1. Подготовить расшифровку

Поместить текст транскрипции в файл, например:

```text
data/ivanova-interview.txt
```

### 2. Настроить LLM-провайдер

Скопировать `.env.example` в `.env` и заполнить нужный раздел.

### 3. Запустить генератор

С Ollama:

```bash
$env:OLLAMA_URL = "http://localhost:11434"
$env:OLLAMA_MODEL = "llama3:latest"
python main.py --input data/ivanova-interview.txt --name "Мария Иванова" --output output/ivanova-strategy.md
```

С Claude:

```bash
python main.py --input data/ivanova-interview.txt --name "Мария Иванова" --output output/ivanova-strategy.md
```

Без LLM (fallback):

```bash
python main.py --input data/ivanova-interview.txt --name "Мария Иванова" --output output/ivanova-strategy.md --fallback
```

### 4. Проверить выходной файл

Открыть `output/ivanova-strategy.md`, проверить:

- имя специалиста;
- УТП;
- цены и услуги;
- кейсы;
- первые шаги.

Всё, что отмечено **[требует проверки]**, нужно дозаполнить вручную или уточнить у специалиста.

## Структура выходного документа

1. **Сводка за 2 минуты** — УТП, отличие, барьер, три первых действия.
2. **Аналитика активов** — что уже есть у специалиста.
3. **Позиционирование** — аудитория, голос, как показывать рынку.
4. **Готовые тексты** — био, услуги, кейсы, контент-план, скрипт первой беседы.
5. **План продвижения** — первые шаги, метрики, резюме.
6. **Контакты продюсера** — подпись в конце документа.

## Ограничения

- Инструмент не делает транскрибацию MP3. MP3 нужно перевести в текст заранее (Whisper, Яндекс SpeechKit, Sonix и т.п.).
- Качество стратегии зависит от качества транскрипции: чем конкретнее ответы специалиста, тем лучше результат.
- Fallback-режим извлекает только базовые сущности (цены, телефон, опыт, кейсы). Для полного анализа нужен LLM.

## Разработка

- Язык: Python 3.10+
- Зависимости: `anthropic`, `openai`, `requests`, `python-dotenv`
- Тестовый запуск: `python main.py --input data/sample-transcript.txt --output output/strategy.md`
